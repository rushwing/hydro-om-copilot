# Hydro O&M Copilot

Hydro O&M Copilot 是一个面向水电机组异常诊断与消缺辅助的智能运维 MVP 项目，服务对象为水电厂运行值班员、检修工程师及相关生产管理人员。

## 版本说明

当前仓库基线版本：`V0.0.1`

当前 MVP 特征：

- 以 RAG 知识增强为主，不涉及自主控制。
- 聚焦水电机组三类高频异常主题。
- 用于原型验证、演示汇报和后续知识库迭代。

## 项目范围

### 当前纳入范围

- 针对水轮发电机组异常文本描述的智能诊断辅助。
- 基于渐进式披露的异常路由与专题诊断。
- 面向内部知识库的规程、规则、案例联合检索。
- 输出症状抽取、根因排序、待确认项、检查步骤、风险提示、班组汇报/缺陷单草稿。
- 预留电厂现场化数据接入能力。

### 当前专题范围

1. 机组振动与摆度异常。
2. 调速器油压与液压控制异常。
3. 轴承温升与冷却水异常。

### V0.0.1 暂不包含

- 保护动作或控制指令的自动执行。
- 与实时历史数据库、SCADA/DCS 的完整在线对接。
- 替代电厂正式运行规程、事故处理规程和调度指令。
- 超出当前三类异常主题的全设备覆盖。
- 对任意监控截图进行高置信度图像原生诊断。

## 项目边界与设计原则

### 核心定位

本项目作为“水电机组异常诊断与消缺辅助 Copilot”是可行的，但必须明确定位为：

- 有证据链的辅助决策系统。
- 面向运行和检修人员的分析辅助系统。
- 不替代值班员判断、不替代保护逻辑、不替代正式规程执行的智能助手。

### 一期可行性判断

一期聚焦的三个场景具备较高可行性：

1. 振动与摆度异常。
2. 调速器油压异常。
3. 轴承温升与冷却水异常。

之所以适合作为 MVP 起点，主要因为：

- 知识边界相对清晰。
- 症状、参数、保护、处置之间的链条比较完整。
- 适合先通过文本驱动的 RAG 方式落地。
- 能在现有语料基础上快速搭建出可演示的一阶段能力。

### 一期生产实际价值

一期最先兑现的价值，不是追求“100% 找准根因”，而是优先解决以下问题：

- 缩短异常首响应用时。
- 统一运行、检修、班组之间的分析口径。
- 减少漏查项和经验遗漏。
- 自动生成班组汇报和缺陷单草稿。
- 沉淀老师傅经验，降低经验依赖带来的波动。

### 风险边界

所有涉及以下动作的建议，必须明确区分“规程硬规则”和“模型推断”：

- 降负荷。
- 停机。
- 保护动作。
- 强制切换。
- 事故处置升级。

对上述内容，系统应始终保留以下约束：

- 规程红线优先于模型判断。
- 通用规则库优先于自由生成结论。
- 厂站正式定值和现场规程优先于通用知识。
- 最终操作必须保留人工确认。

### 图像能力边界

图像能力建议采用保守上线策略：

- 报警截图、OCR 文本提取可优先落地。
- 监控趋势图自动诊断在一期只作为辅助能力，不作为主要结论来源。

原因在于不同电厂之间往往存在较大差异：

- 监控画面模板不统一。
- 曲线标注方式不统一。
- 时标和量纲习惯不统一。
- 截图质量、分辨率和裁切方式差异较大。

因此，当前版本更适合将图像能力定位为“辅助提取和辅助提示”，而不是“主诊断引擎”。

## 一句话目标

当运行或检修人员输入一段机组异常描述，或上传趋势图/报警截图后，系统能够辅助完成：

- 异常症状抽取。
- 规程、规则、案例联查。
- 根因排序。
- 现场检查顺序建议。
- 风险提示。
- 班组汇报与缺陷单草稿生成。

## 渐进式披露设计

当前 L0/L1/L2 文档已经统一补充了以下结构化字段：

- `doc_id`
- `doc_level`
- `knowledge_type`
- `route_keys`
- `upstream_docs`
- `downstream_docs`
- `related_rules`
- `related_cases`
- `site_stub_refs`

这套结构主要用于：

- 先路由、后生成，减少大模型直接自由发挥。
- 在专题范围内做更精确的 RAG 检索。
- 保留“结论来自哪里”的解释链路。
- 为后续转成 JSON、图谱或知识节点做准备。

## L3 现场化层说明

V0.0.1 版本已经预留现场化模板，方便后续导入：

- 机组台账。
- 测点 Tag 映射。
- 保护定值。
- 受限负荷区。
- 历次缺陷。
- 检修履历。
- 厂站规程版本。

当前这些文件仅为模板占位，因此本版本仍然是“通用型水电运维知识助手”，还不是某一座具体电厂、某一台具体机组的专属专家系统。

## 安全定位

本项目定位为“诊断辅助与文稿生成辅助”，不替代：

- 电厂正式规程。
- 继电保护与联锁逻辑。
- 当值人员的操作责任。
- 调度命令与生产指挥流程。

凡涉及降负荷、停机、保护动作、强制切换、事故处置的建议，均应以厂站正式规程、实时工况和人工复核为准。
