# Harness Standard — 知识库文档入库规范

> 本规范约束所有进入 `knowledge_base/docs_internal/` 的 Markdown 文档，
> 保障文档在向量分片后仍能被准确检索，且 LLM 生成时不产生术语歧义。

---

## 1. 适用范围

- **组件**：RAG 知识库 → 分片 → 向量存储 → 混合检索
- **输入类型**：`knowledge_base/docs_internal/**/*.md`
- **触发时机**：
  - [ ] 新增文档（PR 审查时）
  - [ ] 修改标题结构、表格列名或 frontmatter 时
  - [ ] 季度定期 audit

---

## 2. 输入规范

### 2.1 Frontmatter 完整性

| 项目 | 内容 |
|---|---|
| 规则 | 每份文档必须包含 `doc_id`、`doc_level`、`knowledge_type`、`route_keys`；L3 文档额外要求 `upstream_docs` |
| 影响 | 缺 `doc_id` 导致 corpus 过滤失效；缺 `route_keys` 导致症状路由无法命中目标文档 |
| 好示例 | `doc_id: L2.TOPIC.VIB.001` + `route_keys: [vibration_swing, shaft_runout]` |
| 坏示例 | `doc_id: 振动指南`（非标格式）或完全缺失 `route_keys` |

`doc_id` 格式正则：`L[0-3]\.[A-Z]+(\.[A-Z]+)*\.\d{3}`

---

### 2.2 标题层级 = 语义粒度

| 项目 | 内容 |
|---|---|
| 规则 | `##` 下内容控制在 **400–550 字**以内，且单个 `##` 只表达一个独立知识点；禁止空标题 |
| 影响 | 超过 600 字时分片器二次切割，切割点随机，丢失语义完整性；空标题产生无内容 chunk |
| 好示例 | `## 油压持续下降的诊断步骤`（一个流程，<450 字） |
| 坏示例 | `## 油压异常诊断`（内含识别、原因、步骤、案例四个独立知识点，>800 字） |

---

### 2.3 表格列名自解释

| 项目 | 内容 |
|---|---|
| 规则 | 列名在脱离上下文后仍能独立说明含义；禁止使用纯通用词作为唯一列名 |
| 影响 | 每个表格 chunk 携带列名作为 LLM 上下文，列名模糊导致无法推断列的语义，产生幻觉 |
| 好示例 | `\| 监测参数 \| 报警阈值（℃） \| 跳闸阈值（℃） \| 参考规则编号 \|` |
| 坏示例 | `\| 字段 \| 示例 \| 说明 \|` |

**禁用列名及推荐替换：**

| 禁用 | 替换方向 |
|---|---|
| 字段 | 参数名称 / 检查项 / 配置项 |
| 示例 | 示例值 / 典型取值 |
| 说明 | 字段说明 / 单位与约束 / 定义 |
| 备注（作为唯一信息列） | 经验教训 / 标准来源 / 注意事项 |
| 值 | [具体量纲，如"温度（℃）"] |

---

### 2.4 禁止复合列名

| 项目 | 内容 |
|---|---|
| 规则 | 禁止用 `/` 或 `与` 连接两个独立概念作为列名，应拆分为两列 |
| 影响 | LLM 无法判断 `/` 两侧数据的逻辑关系（AND / OR / 举例）；提取结构化信息时出错 |
| 好示例 | `\| 可观察参数 \| 频谱特征 \|`（两列） |
| 坏示例 | `\| 可观察参数/频谱特征 \|`（一列，复合概念） |

---

### 2.5 数值阈值必须带单位

| 项目 | 内容 |
|---|---|
| 规则 | 阈值数值在列名中或单元格中必须标明单位；温度用 ℃，压力用 MPa，振动用 mm（峰峰值）|
| 影响 | LLM 生成报告时若单位缺失会产生错误换算（如把 MPa 当 kPa），产生危险幻觉 |
| 好示例 | `报警阈值（℃）` 列下填 `65` |
| 坏示例 | `告警值` 列下填 `65` —— 单位不明 |

---

### 2.6 案例文档结构（L2.SUPPORT.CASE）

| 项目 | 内容 |
|---|---|
| 规则 | 每个案例用独立 `##` 隔开，必须包含：**故障现象 / 根因 / 临时处置 / 根治措施 / 经验教训**五要素 |
| 影响 | 案例结构一致，确保任意关键词检索命中完整案例，而非缺失根因或经验的碎片 |
| 好示例 | `## 案例：上导瓦温突升（#1机 2023-08）` + 五要素完整 |
| 坏示例 | 案例与分析内容混在同一 `##` 下，或五要素缺失"经验教训" |

---

### 2.7 统一术语（参见附录速查词汇表）

| 项目 | 内容 |
|---|---|
| 规则 | 使用速查词汇表中的标准术语；同一概念禁止在不同文档中使用不同词汇 |
| 影响 | 同义词导致 BM25 检索漏召；向量空间中语义密度稀释，降低混合检索召回率 |
| 好示例 | 统一用"故障现象"描述可观察到的异常表现 |
| 坏示例 | 同一概念在 VIB 指南用"故障表现与症状"，在 BEAR 指南用"异常现象" |

---

## 3. 审查清单

### 自动可检查（脚本 / CI）

- [ ] 所有 `.md` 文件含 `doc_id` frontmatter 字段
- [ ] `doc_id` 格式符合 `L[0-3]\.[A-Z]+(\.[A-Z]+)*\.\d{3}` 正则
- [ ] 无空标题（标题行后紧跟另一标题行，中间无正文或表格）
- [ ] 表格列名不含纯通用词：字段 / 示例 / 值（孤立出现时）
- [ ] 表格列名不含 `/` 连接两个独立概念
- [ ] L3 文档（路径含 `L3_`）含 `upstream_docs` 字段
- [ ] 数值单元格中含阈值的列名或单元格内标明单位（℃ / MPa / mm）

### 人工检查

- [ ] `##` 下内容字数在 400–550 字之间（或为表格）
- [ ] 每个 `##` 只含一个独立知识点
- [ ] 案例文档：每个案例 `##` 含故障现象 / 根因 / 临时处置 / 根治措施 / 经验教训五要素
- [ ] 无速查词汇表外的同义词
- [ ] L3 模板中设备对象名称与台账模板（L3.SITE.001）一致

---

## 4. 验收标准

- **通过**：自动检查全部通过 + 人工检查无高级违规
- **打回修改**：任一自动检查失败
- **打回 + 更新词汇表**：发现新的同义词混用，需先在速查词汇表确定标准项再修改文档

---

## 5. 速查词汇表

> 每行格式：**标准术语**（含义）← 已知禁用同义词列表

| 标准术语 | 含义 | 禁用同义词 |
|---|---|---|
| 故障现象 | 可观察到的异常表现（客观描述） | 异常现象、故障表现与症状、异常表现、异常征兆 |
| 根因 | 经分析确认的最终根本原因 | 根因结论、最终根因、根本原因（保留书面场合） |
| 临时处置 | 快速止损但不消除根因的操作 | 临时措施、应急处置、临时处理 |
| 根治措施 | 消除根因的永久性措施 | 消缺措施、根治方法、永久措施 |
| 工况 | 机组当前运行状态（负荷+水头+温度的组合） | 运行工况、当前工况（可接受修饰词，但基本词统一） |
| 报警阈值 | 触发运行告警的数值边界 | 告警值、预警值、报警值（保留口语场合注释） |
| 跳闸阈值 | 触发保护跳闸的数值边界 | 跳闸值、事故值、保护值 |
| 摆度 | 轴的径向相对位移（单位 μm 或 mm，峰峰值） | 径向位移（混淆时加注"相对"区分）|
| 瓦温 | 轴承导瓦的温度（℃） | 轴承温度（易与油温混淆，禁止混用） |
| 设备对象 | 具体受监测或维修的硬件部件名称 | 处理设备对象、设备名称、部件对象 |
| 故障现象 ↔ 已处理 | 缺陷状态：已消缺且验收合格 | 已消缺（L3.SITE.005 遗留，已统一） |

> **待定义**（扫描到但尚未确认标准项，需人工决策）：
>
> - `严重程度` 枚举值：一般 / 重要 / 紧急 vs. low / medium / critical —— 需确认是否对齐系统风险等级
> - `检修类别`：A修 / B修 / 临修 / 抢修 —— 需确认是否需要展开定义（A修 = 大修周期？）
> - `上位依据`：国标 / 行标 / 集团制度 —— 是否需要规范为标准编号格式（DL/T xxx-xxxx）
