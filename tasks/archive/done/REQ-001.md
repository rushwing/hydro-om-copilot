# Goal

实现基于 Server-Sent Events 的流式诊断接口，让前端能够实时接收诊断过程中的状态更新、token 流和最终结果。

# In Scope

- `POST /diagnosis/run` 端点，`Content-Type: text/event-stream`
- 四类事件：`status`（节点进度）、`token`（LLM 流式 token）、`result`（最终 DiagnosisResult）、`error`（异常）
- SSE 连接的生命周期管理（建立、流式输出、关闭）

# Out of Scope

- 认证鉴权（见 REQ-016）
- 自动诊断触发（见 REQ-009）
- 前端 SSE 消费（见 REQ-005）

# Acceptance Criteria

- `POST /diagnosis/run` 返回 `Content-Type: text/event-stream`
- 至少发出 `status`、`token`、`result` 三类事件
- 异常时发出 `error` 事件并关闭连接
- 正常完成时 SSE 流正常关闭
