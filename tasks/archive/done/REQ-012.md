# Goal

建立持续集成工作流，在 PR 开启/更新时（合并前）自动运行代码质量检查。

# In Scope

- `.github/workflows/agent-loop.yml` 工作流文件
- 触发事件：`pull_request`（`opened` / `synchronize` / `reopened`），目标分支 `main`
- 步骤（按序）：
  1. `uv tool run ruff check backend/`（+ ruff format --check）
  2. `cd frontend && npm run type-check`（tsc --noEmit）
  3. `cd frontend && npm run lint`（ESLint）
  4. `cd frontend && npm run test`（Vitest）
  5. `uv run pytest backend/`
  6. `python3 scripts/check_req_coverage.py --strict`

# Out of Scope

- **GitHub branch protection 配置**：将 workflow job 设为 required status check 是仓库级 GitHub Settings 操作，不在代码提交范围内，需人工在 Settings → Branches 完成
- E2E 测试 CI 集成（见 REQ-013）
- Docker 构建 CI（见 docs/07-build-deploy.md）
- LangSmith 追踪集成（见 REQ-014）
- Claim PR（标题匹配 `^claim:` 的 PR 豁免 required reviews，见 ci-standard.md）

# Acceptance Criteria

- `.github/workflows/agent-loop.yml` 存在，trigger 为 `pull_request` 事件 ✅
- PR 开启后 CI 在合并前自动运行全部 job ✅
- 全部 job 通过时 check 显示绿灯 ✅
- **强制阻断 merge** 需在 GitHub Settings → Branch protection 将 job 设为 required（仓库配置步骤，不在本 REQ 范围）
