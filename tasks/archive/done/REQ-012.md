# Goal

建立持续集成工作流，在 PR 开启/更新时（合并前）自动运行代码质量检查，将其设为 required status check 以阻断不合格代码进入主干。

# In Scope

- `.github/workflows/agent-loop.yml` 工作流文件
- 触发事件：`pull_request`（`opened` / `synchronize` / `reopened`），目标分支 `main`
- 步骤（按序）：
  1. `uv tool run ruff check backend/`
  2. `cd frontend && npx tsc --noEmit`
  3. `cd frontend && npm run lint`
  4. `pytest backend/`
  5. `python3 scripts/check_req_coverage.py --strict`
- GitHub branch protection：将此 workflow job 设为 required status check

# Out of Scope

- E2E 测试（见 REQ-013）
- Docker 构建 CI（见 docs/07-build-deploy.md）
- LangSmith 追踪集成（见 REQ-014）
- Claim PR（标题匹配 `^claim:` 的 PR 豁免 required reviews，见 ci-standard.md）

# Acceptance Criteria

- `.github/workflows/agent-loop.yml` 存在，trigger 为 `pull_request` 事件
- PR 开启后 CI 在合并前自动运行全部步骤
- 任一步骤失败时该 PR 无法 merge（required status check 红灯）
- 全部步骤通过时 check 显示绿灯，不阻断 merge

# Test Case Design Notes

> 需覆盖：全部通过时绿灯不阻断、ruff 报错时红灯阻断、tsc 报错时红灯阻断、pytest 失败时红灯阻断、check_req_coverage --strict 失败时红灯阻断。
