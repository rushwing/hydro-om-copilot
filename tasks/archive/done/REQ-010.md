# Goal

暴露自动诊断结果的查询 API，让前端能够轮询或按需获取最新的自动诊断记录列表。

# In Scope

- `GET /diagnosis/auto-results` 端点：查询结果列表
- `GET /diagnosis/auto/status` 端点：查询 Worker 运行状态
- `POST /diagnosis/auto/start` 端点：启动轮询 Worker
- `POST /diagnosis/auto/stop` 端点：停止轮询 Worker
- `POST /diagnosis/auto/reset-cooldowns` 端点：重置所有机组冷却计时器
- 从 `diagnosis_store` 环形缓冲区读取结果
- 返回 `AutoDiagnosisRecord` 列表（含 `root_causes`、`check_steps`、`report_draft`）
- 支持分页或限制返回条数

# Out of Scope

- 结果推送（SSE/WebSocket，当前为轮询模式）
- 历史持久化到数据库（当前为内存环形缓冲区）

# Acceptance Criteria

- `GET /diagnosis/auto-results` 返回 JSON 列表，每条记录包含 `root_causes`、`check_steps`、`report_draft` 字段
- 环形缓冲区满时返回最新的 N 条记录
- `GET /diagnosis/auto/status` 返回 Worker 当前运行状态
- `POST /diagnosis/auto/start` 启动 Worker，`POST /diagnosis/auto/stop` 停止 Worker
- `POST /diagnosis/auto/reset-cooldowns` 清空所有机组冷却计时器

# Test Case Design Notes

> **结果查询**：空列表、满缓冲区返回最新 N 条、各字段（root_causes / check_steps / report_draft）格式验证。

| Acceptance Criteria | 覆盖状态 | 测试函数 |
|---|---|---|
| GET /auto-results 返回含必要字段的 JSON 列表 | ✅ | `TestAutoResultsEndpoint::test_record_schema_fields_present`（字段级验证）；`TestAutoResults::test_results_returns_list` 仅验证列表形状 |
| 环形缓冲区满时返回最新 N 条 | ✅ | `TestDiagnosisStore` |
| GET /auto/status 返回 Worker 运行状态 | ✅ | `TestAutoStatus` |
| POST /auto/start 启动 Worker（幂等） | ✅ | `TestAutoStart` |
| POST /auto/stop 停止 Worker（幂等） | ✅ | `TestAutoStop` |
| POST /auto/reset-cooldowns 重置冷却 | ✅ | `TestResetCooldowns` |
>
> **Worker 控制语义**（高回归风险，TC 必须覆盖）：
> - `POST /auto/start` 幂等性：Worker 已运行时重复调用不产生第二个 Worker 实例
> - `POST /auto/stop` 语义：停止时当前正在诊断的任务行为（丢弃 or 等待完成），已入队但未开始的任务是否清空
> - `POST /auto/stop` 幂等性：Worker 未运行时调用不报错
> - `POST /auto/reset-cooldowns` 效果：重置后同一 `unit_id` 可立即再次触发诊断（不受 `DIAGNOSIS_COOLDOWN` 拦截）
> - start → stop → start 序列：重启后 Worker 可正常消费队列
