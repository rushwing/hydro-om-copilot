# Goal

暴露自动诊断结果的查询 API，让前端能够轮询或按需获取最新的自动诊断记录列表。

# In Scope

- `GET /diagnosis/auto-results` 端点：查询结果列表
- `GET /diagnosis/auto/status` 端点：查询 Worker 运行状态
- `POST /diagnosis/auto/start` 端点：启动轮询 Worker
- `POST /diagnosis/auto/stop` 端点：停止轮询 Worker
- `POST /diagnosis/auto/reset-cooldowns` 端点：重置所有机组冷却计时器
- 从 `diagnosis_store` 环形缓冲区读取结果
- 返回 `AutoDiagnosisRecord` 列表（含 `root_causes`、`check_steps`、`report_draft`）
- 支持分页或限制返回条数

# Out of Scope

- 结果推送（SSE/WebSocket，当前为轮询模式）
- 历史持久化到数据库（当前为内存环形缓冲区）

# Acceptance Criteria

- `GET /diagnosis/auto-results` 返回 JSON 列表，每条记录包含 `root_causes`、`check_steps`、`report_draft` 字段
- 环形缓冲区满时返回最新的 N 条记录
- `GET /diagnosis/auto/status` 返回 Worker 当前运行状态
- `POST /diagnosis/auto/start` 启动 Worker，`POST /diagnosis/auto/stop` 停止 Worker
- `POST /diagnosis/auto/reset-cooldowns` 清空所有机组冷却计时器
