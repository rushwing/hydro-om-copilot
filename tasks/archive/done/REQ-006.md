# Goal

将诊断会话历史持久化到 localStorage，确保页面刷新后用户仍可访问历史诊断记录。

# In Scope

- `useSessionHistory` hook 的 localStorage 读写逻辑
- `HistoryPage`：会话历史列表渲染页面（含手动诊断与自动诊断历史）
- 历史记录的加载与还原

# Out of Scope

- 服务端历史存储（当前只用 localStorage）
- 历史记录导出/分享
- 跨设备同步

# Acceptance Criteria

- 页面刷新后 `localStorage` 中的会话历史完整还原
- 历史 `DiagnosisResult` 列表字段无数据丢失
- 最多保留最近 N 条记录（防止 localStorage 溢出）

# Test Case Design Notes

> 需覆盖：刷新后历史还原、localStorage 满时的淘汰策略、空历史的初始状态。
