# Goal

实现手动诊断的完整前端界面，用户输入故障描述后触发诊断，通过 SSE 实时展示诊断进度和结果。

# In Scope

- `InputPanel`：故障描述输入、机组选择、提交按钮
- `StreamingOutput`：实时 token 流显示
- `RootCauseCard`：根因分析结果展示
- `ChecklistPanel`：SOP 检查清单
- `ReportDraft`：运维报告草稿
- `RiskBadge`：风险等级徽章（深色主题使用本地 `darkRiskColors`）
- `SourcesPanel`：诊断结果检索来源文档列表
- `DiagnosisPage`：承载上述所有组件的页面容器
- `ManualPage`：手动诊断入口页（导航到 DiagnosisPage）
- Zustand 全局状态（`diagnosisStore`）
- `useSSEDiagnosis` hook 管理 SSE 生命周期

# Out of Scope

- 会话历史持久化（见 REQ-006）
- 自动诊断结果展示（见 REQ-011）
- 登录页（见 REQ-017）

# Acceptance Criteria

- 六个 UI 组件全部渲染无报错
- SSE 流程中 token 实时追加到 StreamingOutput
- result 事件触发后 RootCauseCard/ChecklistPanel/ReportDraft 正确填充
- RiskBadge 在深色背景下文字可见

# Test Case Design Notes

> 需覆盖：正常完整诊断流程、SSE error 事件的错误提示、RiskBadge 四个等级的颜色渲染。
