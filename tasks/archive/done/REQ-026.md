# Goal

将 `npm run e2e` 接入 GitHub Actions CI PR gate，并补充 4 条覆盖
pending-archive 状态管理核心路径的高价值 E2E 用例。

# In Scope

- `frontend/tests/e2e/pending-archive.spec.ts`：4 条用例
  - Test 1：完整 UI 流（SSE → 稍后处理 → 切页 → 待处理 tab 持久化）
  - Test 2：localStorage seed → reload → 状态恢复
  - Test 3：多条 pending 同时保留（不互相覆盖）
  - Test 4：提交归档 → 移入已归档 tab
- `.github/workflows/agent-loop.yml`：新增 `e2e-smoke` job（PR gate）
- `harness/ci-standard.md`：PR Gate E2E 行更新为 ✅ CI job 已接入

# Out of Scope

- 后端 E2E（需要真实 LLM）
- 移动端 / Firefox / WebKit 浏览器覆盖

# Acceptance Criteria

1. `cd frontend && npm run e2e` 输出 6 passed（smoke 2 + pending-archive 4）
2. CI `e2e-smoke` job 出现在 `.github/workflows/agent-loop.yml` PR gate 中
3. `harness/ci-standard.md` PR Gate 表 E2E 行不再有 🔲 标记
