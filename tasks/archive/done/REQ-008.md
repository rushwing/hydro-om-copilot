# Goal

实现跨传感器故障聚合器，周期性轮询三个 FastMCP 伪传感器 Server，当检测到异常读数时生成包含故障摘要的 `FaultSummary` 对象。

# In Scope

- `fault_aggregator.py` 主逻辑
- 轮询三个传感器 Server（MCP 客户端调用）
- 跨传感器故障关联分析
- 生成 `FaultSummary`（含 `unit_id`、`fault_types`、`symptom_text`）
- 冷却期管理（同一机组冷却期内不重复生成）

# Out of Scope

- 自动触发诊断（见 REQ-009）
- 传感器 Server 实现（见 REQ-007）

# Acceptance Criteria

- `FaultAggregator` 轮询三传感器后返回 `FaultSummary` 对象
- `FaultSummary` 包含 `unit_id`、`fault_types`、`symptom_text` 字段
- 冷却期内不重复生成同一机组的 `FaultSummary`

# Test Case Design Notes

> 需覆盖：单传感器故障、多传感器同时故障、冷却期内重复触发的过滤。
