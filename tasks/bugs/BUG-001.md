# 现象描述

手动诊断点击“开始诊断”进入 SSE 流式执行后，故障参数输入面板仍然可以继续操作。
当前仍可切换机组、设备、异常类型，也仍可上传图片。

这会让页面处于“诊断已开始，但输入源仍可变动”的不一致状态，用户难以判断当前结果到底对应哪一组输入。

# 预期行为

一旦手动诊断开始并处于进行中，输入面板的可变字段应整体灰化并禁止交互，至少包括：

- 机组选择
- 设备选择
- 异常类型选择
- 图片上传入口

待本次诊断结束、失败、取消或重置后，相关控件再恢复可操作。

# 复现步骤

1. 打开手动诊断页面
2. 选择任意机组、设备、异常类型，并填写故障描述
3. 点击“开始诊断”
4. 在流式诊断尚未结束时，尝试切换机组 / 设备 / 异常类型，或继续上传图片

# 环境信息

- 分支：main
- 相关 commit：待修复时补充

# 根因分析

chipBase 样式缺少 `disabled:opacity-40 disabled:cursor-not-allowed`，导致按钮虽 disabled 但视觉无反馈；图片拖拽区 onDrop/onDragOver 未检查 isRunning，拖拽路径未封堵；图片预览的删除按钮缺少 disabled 属性。

# 修复方案

1. chipBase 加 `disabled:opacity-40 disabled:cursor-not-allowed`，按钮视觉灰化与 disabled 属性同步。
2. handleDrop / handleDragOver 头部加 `if (isRunning) return`，封堵拖拽上传路径。
3. 图片预览的删除按钮加 `disabled={isRunning}`。
4. 拖拽区 className 根据 isRunning 独立分支，显示 `cursor-not-allowed opacity-40`。
