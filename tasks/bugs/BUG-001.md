# 现象描述

手动诊断点击“开始诊断”进入 SSE 流式执行后，故障参数输入面板仍然可以继续操作。
当前仍可切换机组、设备、异常类型，也仍可上传图片。

这会让页面处于“诊断已开始，但输入源仍可变动”的不一致状态，用户难以判断当前结果到底对应哪一组输入。

# 预期行为

一旦手动诊断开始并处于进行中，输入面板的可变字段应整体灰化并禁止交互，至少包括：

- 机组选择
- 设备选择
- 异常类型选择
- 图片上传入口

待本次诊断结束、失败、取消或重置后，相关控件再恢复可操作。

# 复现步骤

1. 打开手动诊断页面
2. 选择任意机组、设备、异常类型，并填写故障描述
3. 点击“开始诊断”
4. 在流式诊断尚未结束时，尝试切换机组 / 设备 / 异常类型，或继续上传图片

# 环境信息

# 根因分析

> 修复者填写；重点检查 `InputPanel`、诊断进行中状态来源，以及图片上传组件是否统一受同一 disabled / loading 状态控制。

# 修复方案

> 修复者填写；建议统一由“诊断进行中”状态驱动整个输入面板的 disabled 语义，避免只锁按钮、不锁其他输入控件。
