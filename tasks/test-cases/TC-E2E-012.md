## 测试目标

验证自动诊断面板在收到 `/diagnosis/auto/status` 轮询响应后，正确渲染 Epoch 编号、待处理队列信息和已完成结果列表。

## 前置条件

- `GET **/diagnosis/auto/status` → mock 运行中状态：
  ```json
  {
    "running": true,
    "is_simulated": true,
    "current": {"unit_id": "#3机", "fault_type": "bearing_temp_cooling"},
    "pending_queue": ["#4机"],
    "completed_count": 2,
    "unit_cooldowns": {},
    "epoch_num": 5,
    "epoch_elapsed_s": 120,
    "epoch_phase": "NORMAL"
  }
  ```
- `GET **/diagnosis/auto-results` → mock 含 1 条已完成结果
- 标准其他 mock

## 操作步骤

1. 访问包含自动诊断面板的页面（`/` 或专属路由）
2. 等待面板初始渲染
3. 断言 Epoch 编号（`5`）可见
4. 断言当前处理机组（`#3机`）可见
5. 断言待处理队列（`#4机`）可见
6. 断言已完成结果列表有 1 条

## 预期结果 / 不应发生

- 所有状态字段在面板中正确显示
- 不应发生：面板显示"未运行"、Epoch 信息缺失
