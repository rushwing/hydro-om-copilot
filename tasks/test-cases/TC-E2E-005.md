## 测试目标

验证 `addToPending` 去重逻辑：已存在 pending 条目的情况下，新诊断结果点击"稍后处理"后，旧条目不被覆盖，两条均留在 pending 列表。

## 前置条件

- localStorage seed：`#1机`（id: `seed-001`）已在 pending 列表
- `POST **/diagnosis/run` → mock 完整 SSE（session_id: `test-session-002`, unit_id: `#2机`）
- 标准 API mock

## 操作步骤

1. 访问 `/`
2. 在 textarea 中填写 `#2机振动超限`
3. 点击"开始诊断"
4. 等待"稍后处理"按钮出现（超时 10s）
5. 点击"稍后处理"
6. 访问 `/history`
7. 断言 `#1机` 可见
8. 断言 `#2机` 可见

## 预期结果 / 不应发生

- 待处理 tab 同时显示 `#1机` 和 `#2机`
- 不应发生：旧条目被覆盖、只有 `#2机` 存在
