## 测试目标

验证后端返回 SSE `error` 事件时，UI 正确展示错误提示信息，且该次诊断不被写入 pending 或 archived 列表。

## 前置条件

- `POST **/diagnosis/run` → mock SSE，包含：
  - `event: error` + `data: {"message":"LLM 调用超时"}`
  - `event: done` + `data: {}`
- localStorage 初始为空
- 标准其他 mock

## 操作步骤

1. 访问 `/`
2. 填写症状描述并点击"开始诊断"
3. 等待错误状态渲染
4. 断言页面显示错误提示文本（如含"超时"或"错误"）
5. 访问 `/history`
6. 断言待处理 tab 和已归档 tab 均为空

## 预期结果 / 不应发生

- 错误提示可见
- history 页两个 tab 均无条目
- 不应发生：错误状态下"稍后处理"按钮可点击、错误结果被写入 localStorage
