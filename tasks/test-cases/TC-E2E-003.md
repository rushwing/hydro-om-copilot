## 测试目标

验证用户点击"稍后处理"后，诊断结果写入 localStorage pending 列表，且跨页导航后仍可在历史页待处理 tab 中看到对应机组。

## 前置条件

- `GET **/health` → mock ok
- `GET **/diagnosis/auto/status` → mock idle 状态
- `GET **/diagnosis/auto-results` → mock `[]`
- `POST **/diagnosis/run` → mock 完整 SSE（session_id: `test-session-001`, unit_id: `#1机`）
- localStorage 初始为空

## 操作步骤

1. 访问 `/`
2. 在 textarea（`placeholder*="描述异常现象"`）中填写 `#1机振动异常，导叶开度反馈偏差`
3. 点击"开始诊断"按钮
4. 等待"稍后处理"按钮出现（超时 10s）
5. 点击"稍后处理"
6. 访问 `/history`
7. 断言 `#1机` 文本可见
8. 访问 `/`，再访问 `/history`
9. 再次断言 `#1机` 文本可见

## 预期结果 / 不应发生

- 两次导航后 `/history` 均能看到 `#1机`
- 不应发生：状态丢失、pending 列表清空、tab 默认不在待处理
