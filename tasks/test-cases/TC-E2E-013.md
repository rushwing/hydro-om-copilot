## 测试目标

验证自动诊断结果在超过未查看超时阈值后自动移入待处理列表；而用户正在查看（expanded）的条目不触发自动归档。

## 前置条件

- localStorage seed：两条 auto 结果，其中：
  - 条目 A：`triggered_at` 超过超时阈值（如 10 分钟前），未展开
  - 条目 B：`triggered_at` 超过超时阈值，当前处于展开/查看状态
- 自动归档超时阈值通过 mock 或测试配置缩短（如 1s），或直接 seed 足够旧的时间戳
- 标准 API mock

## 操作步骤

1. 访问 `/history`（或触发超时检查的路由）
2. 等待自动归档逻辑运行
3. 切换至"待处理"tab
4. 断言条目 A 在待处理列表中
5. 断言条目 B 不在待处理列表（仍在查看态）

## 预期结果 / 不应发生

- 条目 A 自动归入待处理
- 条目 B 不被自动归档（查看中豁免）
- 不应发生：所有超时条目被归档忽略、查看中条目被误归档
