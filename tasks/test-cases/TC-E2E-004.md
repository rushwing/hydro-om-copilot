## 测试目标

验证 localStorage 中已存在的 pending 条目在页面硬刷新后仍能被正确读取并显示，即持久化读取路径无回归。

## 前置条件

- `addInitScript` 在应用 JS 加载前写入 localStorage：
  ```json
  [{"id":"seed-001","unit_id":"#1机","fault_types":["振动与摆度"],"risk_level":"medium","root_causes":[],"check_steps":[],"report_draft":null,"triggered_at":"2026-03-15T08:00:00.000Z","archived_at":"2026-03-15T08:05:00.000Z","source":"auto_diagnosed","completed":false}]
  ```
  key: `hydro_om_pending_archive`
- 标准 API mock（health / auto-status / auto-results）

## 操作步骤

1. 访问 `/history`
2. 断言 `#1机` 可见
3. 执行 `page.reload()`
4. 再次断言 `#1机` 可见

## 预期结果 / 不应发生

- 刷新前后均显示 `#1机`
- 不应发生：刷新后 pending 列表清空、条目消失
