## 测试目标

验证用户在 HumanNotes 中填写运维备注后提交归档，条目进入已归档 tab，并在页面刷新后持久保留（localStorage 写入正确）。

## 前置条件

- localStorage seed：`#1机`（`completed: false`）已在 pending 列表
- 标准 API mock

## 操作步骤

1. 访问 `/history`
2. 展开 `#1机` 卡片
3. 在备注输入框填写文本（如 `已检查调速器油压，正常`）
4. 点击"提交归档"
5. 切换至"已归档"tab，确认 `#1机` 可见
6. 执行 `page.reload()`
7. 再次确认 `#1机` 在已归档 tab 中仍可见

## 预期结果 / 不应发生

- 刷新后已归档条目仍在
- 不应发生：刷新后已归档列表清空、备注内容丢失
