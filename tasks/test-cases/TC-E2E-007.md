## 测试目标

验证用户手动提交诊断后，完整 SSE 流程结束时所有关键 UI 组件均正确渲染：根因列表、检查步骤、风险等级徽章、"稍后处理"与"提交归档"按钮。

## 前置条件

- `POST **/diagnosis/run` → mock 完整 SSE，包含：
  - `status` (symptom_parser start/end)
  - `result` 事件：`root_causes` 至少 1 条、`check_steps` 至少 1 条、`risk_level: "high"`
  - `done` 事件
- 标准 health / auto-status / auto-results mock

## 操作步骤

1. 访问 `/`
2. 填写症状描述
3. 点击"开始诊断"
4. 等待 SSE 完成（`done` 事件）
5. 断言根因列表可见（≥1 条）
6. 断言检查步骤可见（≥1 条）
7. 断言风险等级徽章显示 `高风险` / `high`
8. 断言"稍后处理"按钮可见
9. 断言"提交归档"按钮可见

## 预期结果 / 不应发生

- 所有上述元素在 SSE 完成后均可见
- 不应发生：页面停留在 loading 状态、result 事件后组件未渲染、按钮不可见
