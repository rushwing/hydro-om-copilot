## 测试目标

验证调用自动诊断 stop 接口后，响应中的 `dropped_queue` 条目被正确写入 pending 待处理列表，且页面刷新后持久保留。

## 前置条件

- `POST **/diagnosis/auto/stop` → mock 响应：
  ```json
  {"status": "ok", "dropped_queue": [{"unit_id": "#5机", "fault_type": "vibration_swing"}]}
  ```
- 标准其他 API mock
- localStorage 初始为空

## 操作步骤

1. 访问包含自动诊断控制面板的页面
2. 点击"停止"按钮（触发 stop 请求）
3. 访问 `/history`，切换至待处理 tab
4. 断言 `#5机` 在待处理列表中
5. 执行 `page.reload()`
6. 再次断言 `#5机` 在待处理列表中

## 预期结果 / 不应发生

- stop 后 `dropped_queue` 条目写入 pending
- 刷新后条目仍在
- 不应发生：`dropped_queue` 被忽略、刷新后丢失
