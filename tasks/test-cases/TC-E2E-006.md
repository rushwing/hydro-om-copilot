## 测试目标

验证从待处理 tab 展开卡片 → 点击"提交归档" → 条目从 pending 移除并出现在已归档 tab，即归档状态转换正确。

## 前置条件

- localStorage seed：`#1机`（id: `seed-001`，`completed: false`）
- 标准 API mock

## 操作步骤

1. 访问 `/history`
2. 点击第一个"查看详情"按钮展开卡片
3. 点击"提交归档"按钮（HumanNotes 组件内）
4. 点击"已归档"tab 按钮
5. 断言 `#1机` 在已归档 tab 中可见

## 预期结果 / 不应发生

- 已归档 tab 中显示 `#1机`
- 不应发生：条目同时出现在待处理和已归档、归档后 pending tab 仍有该条目
