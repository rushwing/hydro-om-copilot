## 测试目标

验证来自三种不同来源（手动诊断 `manual`、自动诊断 `auto_diagnosed`、stop dropped `dropped`）的 pending 条目，完成归档操作后均正确移入已归档列表，无论来源差异。

## 前置条件

- localStorage seed：三条 pending 条目，`source` 分别为 `manual`、`auto_diagnosed`、`dropped`
  - 条目 M：`unit_id: "#1机"`, `source: "manual"`
  - 条目 A：`unit_id: "#2机"`, `source: "auto_diagnosed"`
  - 条目 D：`unit_id: "#3机"`, `source: "dropped"`
- 标准 API mock

## 操作步骤

1. 访问 `/history`，待处理 tab 确认三条均可见
2. 展开 `#1机` 卡片 → 点击"提交归档"
3. 展开 `#2机` 卡片 → 点击"提交归档"
4. 展开 `#3机` 卡片 → 点击"提交归档"
5. 切换至"已归档"tab
6. 断言 `#1机`、`#2机`、`#3机` 均在已归档列表中

## 预期结果 / 不应发生

- 三条来源不同的条目均成功归档
- 不应发生：特定来源的条目归档失败、已归档 tab 只显示部分来源
