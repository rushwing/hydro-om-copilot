# Goal

在每次诊断触发时记录审计日志，包含操作用户、会话 ID 和时间戳，满足运维合规要求。

# In Scope

- 审计日志中间件或依赖注入
- 每次 `POST /diagnosis/run` 触发时写入审计记录
- 审计字段：`user_id`、`session_id`、`timestamp`、`unit_id`、`action`
- 审计日志查询接口（`GET /audit/logs`，管理员专用）

# Out of Scope

- 多租户日志隔离（见 REQ-021）
- 日志导出/合规报告（Phase-006 后续）

# Acceptance Criteria

- 每次诊断触发后审计记录写入
- 记录包含 `user_id`、`session_id`、`timestamp` 字段
- `GET /audit/logs` 管理员可查询历史审计记录

# Test Case Design Notes

> 需覆盖：诊断触发后日志写入验证、非管理员访问 /audit/logs 返回 403、日志字段完整性校验。

# Agent Notes

依赖 REQ-016 JWT 认证，从 token 中提取 user_id。
