# Goal

集成 LangSmith 可观测性平台，在诊断 Pipeline 执行时自动上报节点执行记录，并将 `run_id` 透传给前端，支持诊断溯源。

# In Scope

- LangSmith SDK 配置（`LANGCHAIN_TRACING_V2`、`LANGCHAIN_API_KEY` 环境变量）
- LangGraph 节点执行自动上报到 LangSmith
- `run_id` 注入 SSE `result` 事件
- `LANGCHAIN_TRACING_V2=false` 时完全无副作用

# Out of Scope

- LangSmith eval 接入（Phase-003 后续）
- 前端溯源面板（见 REQ-015）
- 生产环境 LangSmith 账号配置

# Acceptance Criteria

- `LANGCHAIN_TRACING_V2=true` 时，LangSmith 项目中可查到对应的诊断 run
- SSE `result` 事件包含 `run_id` 字段
- `LANGCHAIN_TRACING_V2=false` 时，`run_id` 字段为 `null`，无额外开销

# Test Case Design Notes

> 需覆盖：tracing 开启时 run_id 非空、tracing 关闭时 run_id 为 null、LangSmith 连接失败时诊断不中断。

# Agent Notes

详见 `docs/05-langsmith-integration.md`。`LANGCHAIN_TRACING_V2=false` 为生产默认值。
