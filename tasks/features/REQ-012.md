# Goal

建立持续集成工作流，确保每次 PR 合并前自动运行代码质量检查（ruff lint、tsc 类型检查、pytest 单元测试），阻断不合格代码进入主干。

# In Scope

- `.github/workflows/agent-loop.yml` 工作流文件
- 触发条件：PR 合并到 `main`
- 步骤：`uv tool run ruff check backend/`、`cd frontend && npx tsc --noEmit`、`pytest backend/`
- 失败时阻断 PR merge

# Out of Scope

- E2E 测试（见 REQ-013）
- Docker 构建 CI（见 docs/07-build-deploy.md）
- LangSmith 追踪集成（见 REQ-014）

# Acceptance Criteria

- `.github/workflows/agent-loop.yml` 文件存在于 repo
- PR 提交后 CI 自动运行 ruff + tsc + pytest
- 任一步骤失败时 PR merge 被阻断

# Test Case Design Notes

> 需覆盖：全部通过时 CI 绿灯、ruff 报错时 CI 失败、tsc 报错时 CI 失败、pytest 失败时 CI 失败。
