# Goal

实现多租户数据隔离机制，确保不同租户（电厂）的诊断记录和知识库数据相互独立，不可跨租户访问。

# In Scope

- 租户 ID 从 JWT token 中提取
- 诊断记录按 `tenant_id` 隔离存储
- ChromaDB/向量存储按租户命名空间隔离
- 跨租户访问返回 403

# Out of Scope

- 租户管理 UI（见 REQ-022）
- 知识库入库多租户支持（后续）

# Acceptance Criteria

- 租户 A 无法读取租户 B 的诊断记录
- 跨租户访问返回 403
- 同一租户内数据访问正常

# Test Case Design Notes

> 需覆盖：同租户数据访问成功、跨租户访问返回 403、无 tenant_id 时的降级处理。

# Agent Notes

依赖 REQ-016 JWT 认证从 token 提取 tenant_id。向量存储需评估 ChromaDB collection 级别隔离方案。
