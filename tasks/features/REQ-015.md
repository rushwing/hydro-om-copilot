# Goal

在前端添加诊断溯源面板，展示 LangGraph 各节点的执行耗时和检索到的 top-3 相关文档，帮助运维人员理解诊断依据。

# In Scope

- 溯源面板组件（展示节点名、执行耗时）
- 检索文档 top-3 展示（文档标题、相关度得分）
- 仅在 `run_id` 非 `null` 时渲染面板
- `LANGCHAIN_TRACING_V2=false` 时面板完全不渲染

# Out of Scope

- LangSmith 追踪集成（见 REQ-014）
- 完整 LangSmith run 链接跳转（可选增强）

# Acceptance Criteria

- 面板展示诊断 Pipeline 各节点名称和执行耗时
- 面板展示检索阶段返回的 top-3 文档（含标题和得分）
- `run_id` 为 `null` 时面板不渲染

# Test Case Design Notes

> 需覆盖：有 run_id 时面板正常渲染、run_id 为 null 时面板不渲染、节点数据缺失时的降级显示。

# Agent Notes

依赖 REQ-014 将 run_id 透传至前端 result 事件。详见 `docs/05-langsmith-integration.md`。
