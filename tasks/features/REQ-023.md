# Goal

实现基于诊断结果的工单自动生成与派发 API，将根因分析和检查步骤转化为可追踪的运维工单。

# In Scope

- `POST /workorder/dispatch`：基于 `session_id` 或 `DiagnosisResult` 生成工单
- 工单字段：`workorder_id`、`unit_id`、`fault_types`、`check_steps`、`status`、`session_id`
- `GET /workorder/{workorder_id}`：查询工单状态
- `PATCH /workorder/{workorder_id}`：更新工单状态（已处理/关闭）
- 工单与 `session_id` 双向关联

# Out of Scope

- 工单派发到外部系统（CMMS 集成，Phase-007 后续）
- 工单移动端 UI（见 REQ-024）

# Acceptance Criteria

- `POST /workorder/dispatch` 返回 `workorder_id`
- 工单包含来自 DiagnosisResult 的 `check_steps` 和 `fault_types`
- `GET /workorder/{workorder_id}` 返回工单当前状态

> 需覆盖：基于 session_id 创建工单、无效 session_id 返回 404、工单状态流转（待处理→处理中→已关闭）。
