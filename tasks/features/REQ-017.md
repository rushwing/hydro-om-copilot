# Goal

实现前端登录页面和路由守卫，确保未登录用户无法访问诊断功能，完成认证后跳转到主诊断页。

# In Scope

- 登录页组件（用户名/密码输入、提交按钮、错误提示）
- 调用 `POST /auth/login` 获取 JWT
- JWT 存储（`localStorage` 或内存）
- React Router 路由守卫（未登录时重定向到登录页）
- Token 过期检测与自动刷新

# Out of Scope

- JWT 鉴权后端（见 REQ-016）
- 用户管理界面（见 REQ-022）

# Acceptance Criteria

- 登录页提交有效凭据后跳转到诊断主页
- 未登录直接访问诊断页时重定向到登录页
- 登录失败时展示错误提示

# Test Case Design Notes

> 需覆盖：有效凭据登录跳转、无效凭据错误提示、未登录访问守卫重定向、token 过期自动刷新。
