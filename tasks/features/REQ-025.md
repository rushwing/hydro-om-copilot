# Goal

为 `tasks/bugs/BUG-*.md` 文档添加机器可执行的 frontmatter 校验器，
并将其接入 CI PR gate，确保 BUG 文档的元数据规范与 `harness/bug-standard.md` §9 一致。

关联 GitHub issue：#36

# In Scope

- `scripts/check_bug_frontmatter.py`：校验脚本
  - 必填字段存在性检查（`bug_id / title / status / severity / priority / owner / related_req / related_tc / reported_by`）
  - `status` 枚举：`open / confirmed / in_progress / fixed / regressing / closed / wont_fix`
  - `severity` 枚举：`S1 / S2 / S3 / S4`
  - `priority` 枚举：`P0 / P1 / P2 / P3`
  - `status == fixed` 时 `related_tc` 非空
  - `status == in_progress` 时 `owner != unassigned`
  - `related_req` 中每个 REQ-ID 在 `tasks/` 目录中存在
  - `depends_on` 中每个 REQ-ID / BUG-ID 在 `tasks/` 目录中存在
  - `--strict` 模式：有任意错误时 `exit(1)`
- `.github/workflows/agent-loop.yml`：新增 `bug-frontmatter` job，在 PR gate 中运行
- `harness/bug-standard.md` §9：将"自动可检查"列表标注为已实现

# Out of Scope

- BUG 内容质量检查（复现步骤、根因分析等人工项）
- BUG 与测试用例文件的双向覆盖扫描

# Acceptance Criteria

1. `python3 scripts/check_bug_frontmatter.py --strict` 在 `tasks/bugs/` 为空时正常退出 0
2. 将一个违规 BUG 文档放入 `tasks/bugs/`，`--strict` 退出 1 并打印具体字段错误
3. CI `bug-frontmatter` job 出现在 `.github/workflows/agent-loop.yml` PR gate 中
