# Goal

实现面向移动端的工单追踪界面，让现场运维人员可在手机上查看诊断工单详情、确认处理状态，并追溯到原始诊断会话。

# In Scope

- 响应式工单列表页（≥375px 宽度适配）
- 工单详情页：展示故障类型、检查步骤、根因分析
- 确认处理操作按钮（更新工单状态）
- 工单与 `session_id` 的双向关联展示（可跳转到原诊断会话）
- Tailwind CSS 移动端响应式样式

# Out of Scope

- 工单创建（见 REQ-023）
- 原生 App（Web 响应式即可）
- 离线模式

# Acceptance Criteria

- 工单列表在 375px 宽度设备上可正常使用
- 点击工单可查看完整检查步骤和根因分析
- 确认处理后工单状态更新
- 工单页面可跳转到关联的诊断会话

# Test Case Design Notes

> 需覆盖：375px 宽度正常渲染、工单状态确认更新、session_id 关联跳转、空工单列表。
