# Goal

为 FastAPI 应用添加基于 JWT 的身份认证，保护诊断 API，并提供登录和 token 刷新接口。

# In Scope

- `POST /auth/login`：用户名/密码验证，返回 `access_token` 和 `refresh_token`
- `POST /auth/refresh`：刷新 access token
- JWT 中间件：未携带有效 token 的请求返回 401
- FastAPI `Depends` 注入认证逻辑
- 环境变量：`JWT_SECRET_KEY`、`JWT_ALGORITHM`、`ACCESS_TOKEN_EXPIRE_MINUTES`

# Out of Scope

- 用户管理 UI（见 REQ-022）
- 多租户隔离（见 REQ-021）
- 审计日志（见 REQ-018）

# Acceptance Criteria

受保护的诊断路由集（以下全部）无有效 token 时返回 401：
- `POST /diagnosis/run`
- `GET /diagnosis/auto-results`
- `GET /diagnosis/auto/status`
- `POST /diagnosis/auto/start`
- `POST /diagnosis/auto/stop`
- `POST /diagnosis/auto/reset-cooldowns`

鉴权接口：
- `POST /auth/login` 以有效凭据返回 `access_token` + `refresh_token`
- `POST /auth/refresh` 以有效 refresh_token 返回新 access_token
- `/health` 保持公开（无需 token）

# Test Case Design Notes

> 需覆盖：
> - 诊断路由集（全部 6 条）无 token → 401
> - 诊断路由集有效 token → 正常响应
> - 过期 access_token → 401
> - 无效签名 token → 401
> - refresh_token 刷新成功 → 返回新 access_token
> - 使用已撤销 refresh_token → 401（若实现了撤销机制）
